**Step 1 — Update AppDbContext.cs.** Replace the current file content with the following (adds `DbSet<Spool>` and `OnModelCreating` override):

```csharp
using Microsoft.EntityFrameworkCore;

public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    public DbSet<Owner> Owners => Set<Owner>();
    public DbSet<Spool> Spools => Set<Spool>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.Entity<Spool>()
            .HasOne(s => s.Owner)
            .WithMany()
            .HasForeignKey(s => s.OwnerId)
            .OnDelete(DeleteBehavior.Restrict);
    }
}
```

Critical: `DeleteBehavior.Restrict` is mandatory (D-03). EF Core's default for required FK is `Cascade` — without this explicit override, deleting an Owner would cascade-delete their Spools, which violates the 409-error requirement.

**Step 2 — Generate the migration.** Run from the `FilamentCatalog/` project directory:
```
dotnet ef migrations add AddSpools
```
