## Spacing Scale

Declared values — all tokens already defined in `app.css :root`:

| Token | Value | Usage |
|-------|-------|-------|
| --space-xs | 4px | Icon gaps, badge padding, inline micro-spacing |
| --space-sm | 8px | Spool row padding, filter bar gap, owner-header internal gap |
| --space-md | 16px | Section padding, modal padding, filter bar horizontal padding |
| --space-lg | 24px | Sidebar internal padding (top/bottom), layout section breaks |
| --space-xl | 32px | Gap between sidebar and spool column in main layout |

Exceptions:
- Sidebar fixed width: 240px (D-05, not a spacing token — structural dimension)
- Sticky header height: 48px (existing, unchanged)
- Owner group header: no additional padding beyond existing --space-sm / --space-md rhythm
- Chevron touch target: minimum 32px height (existing `.chip` height pattern, reused)

## Layout Contract

Source: D-01 through D-05.

### Two-Column Wrapper

```
<div id="main-layout">
  <aside id="balance-sidebar">…</aside>
  <div id="spool-column">
    <div id="filter-bar">…</div>
    <main id="spool-list">…</main>
  </div>
</div>
```

- `#main-layout`: `display: flex; align-items: flex-start;` — starts immediately below `#summary-bar`
- `#balance-sidebar`: `width: 240px; flex-shrink: 0; position: sticky; top: 48px; align-self: flex-start;` — sticky relative to scroll, top offset matches header height
- `#spool-column`: `flex: 1; min-width: 0; overflow-y: auto;` — independent scroll

### Sidebar

- Semantic element: `<aside id="balance-sidebar">` (D-DISCRETION: aside chosen over nav — it contains supplementary content, not navigation)
- Surface: `background: var(--color-surface); border: 1px solid var(--color-border); border-radius: 8px; margin: var(--space-md);`
- Heading: `<h2>Balance Overview</h2>` — 14px, weight 600, padding --space-sm --space-md, border-bottom 1px solid var(--color-border)
- Content: `<table id="balance-table">` rendered directly — no `<details>` wrapper (D-05)
- The `<details id="balance-section">` element in index.html is removed and replaced by `<aside id="balance-sidebar">`
- `#balance-table` styles are unchanged — same `th`/`td` padding, `#balance-table th`, `#balance-table td` rules remain

### Filter Bar

- Constrained to `#spool-column` — does NOT span full page width (D-02)
- `#filter-bar` moves inside `#spool-column`, above `#spool-list`
- Existing filter bar CSS (`#filter-bar`) is unchanged except it no longer sits at body level
- Expand/Collapse All button added to filter bar right side, immediately left of Add Spool button (D-09)
  - Class: `btn-secondary`
  - ID: `#expand-collapse-btn`
  - Label: "Collapse all" (default — all groups expanded on load) / "Expand all"
  - Width: auto (label-driven, not fixed)

### Owner Group Headers

Source: D-06.

```html
<div class="owner-group" data-owner-id="{id}">
  <div class="owner-group-header" role="button" aria-expanded="true" tabindex="0">
    <span class="owner-chevron">▼</span>
    <span class="owner-group-name">{name via textContent}</span>
    <span class="badge owner-spool-count">{n} spools</span>
  </div>
  <div class="owner-group-rows">
    <!-- .spool-row elements -->
  </div>
</div>
```

- `.owner-group-header`: `display: flex; align-items: center; gap: var(--space-sm); padding: var(--space-sm) var(--space-md); background: var(--color-bg); border-bottom: 1px solid var(--color-border); cursor: pointer;`
- `.owner-chevron`: Unicode `▼` (expanded) / `▶` (collapsed) — rendered via textContent. Color: `var(--color-muted)`. Font-size: 12px.
- `.owner-group-name`: 14px, weight 600, `flex: 1`
- `.owner-spool-count` badge: reuses existing `.badge` class + new modifier `.badge-count` — `background: var(--color-bg); color: var(--color-muted); border: 1px solid var(--color-border);`
- Collapsed state: `.owner-group-rows` gets `hidden` attribute (display: none) — consistent with `.spool-row[hidden]` pattern
- Keyboard: `role="button"`, `tabindex="0"`, responds to `Enter` and `Space`

Spool rows inside a group:
- Reuse existing `.spool-row` class unchanged
- Remove `.spool-owner` column from individual rows (owner is now the group header — no redundancy)
- Indent: `padding-left: 32px` on `.spool-row` inside tree view (overrides base --space-md to create visual nesting)

### Owner Filter Interaction

Source: D-07, D-08.

- Selecting an owner in `#filter-owner`: hide all `.owner-group` elements whose `data-owner-id` does not match; show only the matching group
- Groups with zero visible `.spool-row` elements (after filter application) receive `hidden` attribute on `.owner-group` — no empty group headers shown (D-08)
- "All owners" restores all groups (remove `hidden`)

### Collapse State Persistence

Source: D-10, D-11, D-12.

- Default on load: all groups expanded
- localStorage key scheme: `fc:collapse:{ownerId}` — prefix `fc:collapse:` namespaces under the app (fc = filament catalog)
- Value: `"1"` (collapsed) / key absent or `"0"` (expanded)
- On load: read each owner's key; missing key = expanded (D-12)
- On toggle: write key immediately after state change

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Add Spool" (unchanged) |
| Expand/collapse toggle — all expanded | "Collapse all" |

| Destructive action button label | "Confirm Delete" (unchanged from existing) |

Source: empty state and error copy derived from existing `.empty-state` and `.error-banner` patterns in app.css; new copies follow the same tone (imperative, lowercase first word except proper nouns).
