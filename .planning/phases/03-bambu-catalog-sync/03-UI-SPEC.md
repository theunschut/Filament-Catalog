## Color

All tokens already declared in app.css :root. No new colors introduced in this phase.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | --color-bg #f5f5f5 | Page background |
| Secondary (30%) | --color-surface #ffffff | Header, filter bar, spool rows, modal surfaces |
| Accent (10%) | --color-accent #2563eb | "Sync Bambu catalog" button, "Add Spool" button (re-enabled state), focus outlines, .chip.active |
| Destructive | --color-destructive #dc2626 | Delete Spool actions only — not used for sync errors |
| Muted | --color-muted #6b7280 | Stat labels, gear icon, disabled-state text |
| Warning | --color-warning #f59e0b | Balance rows with missing price (.warning-icon) |
| Border | --color-border #e2e8f0 | All borders, dividers |

Accent reserved for:
- "Sync Bambu catalog" button (idle + hover state — same .btn-primary class as Add Spool)
- "Add Spool" button re-enabled state (same .btn-primary)
- Active filter chips (.chip.active)
- Focus outlines (outline: 2px solid var(--color-accent))

Disabled state for "Add Spool" button: `opacity: 0.5; cursor: not-allowed;` — the accent color is muted by opacity, not replaced. This follows the least-surprise principle for native button[disabled].

Pre-sync inline banner (#catalog-empty-notice) uses the existing .error-banner visual style (background #fee2e2, border #fca5a5, color #991b1b) re-purposed as a warning — OR a new .info-banner class (background #dbeafe, border #93c5fd, color #1e40af) that mirrors the sealed-badge palette. Use .info-banner (blue tones) to avoid false alarm connotations from the red error style. Define as a new CSS class in app.css.

Sync-in-progress indicator: The "Last synced" stat value shows "Syncing…" in --color-muted (#6b7280) while polling. No spinner element required — text change is sufficient. Once sync completes the value updates to the formatted timestamp in the normal #1f2937 body color.

Source: app.css :root and component styles, 03-CONTEXT.md D-05, D-06.

## Component Inventory

### New HTML elements required

#### 1. Sync button in sticky header

Location: `#summary-bar` — appended to `.summary-left` after the last stat cell, before `#manage-owners-btn`.

```html
<button id="sync-catalog-btn" class="btn-secondary" type="button" aria-label="Sync Bambu catalog">
  Sync Bambu catalog
</button>
<div class="stat-cell" id="sync-stat">
  <span class="stat-label">Last synced</span>
  <span class="stat-value" id="stat-last-synced">Never</span>
</div>
```

The sync button uses `.btn-secondary` (not `.btn-primary`) so accent color is reserved for the primary "Add Spool" action. During sync: button shows "Syncing…" text and is `disabled`. On error: button returns to "Sync Bambu catalog" and `#stat-last-synced` shows "Sync failed" in --color-destructive.

Source: 03-CONTEXT.md D-06, existing .stat-cell pattern.

#### 2. Pre-sync inline banner in filter bar

Location: Inside `#filter-bar`, inserted before `#add-spool-btn`. Hidden by default; shown when catalog count = 0.

```html
<p id="catalog-empty-notice" class="info-banner" hidden>
  Sync the Bambu catalog first to enable adding spools.
</p>
```

New `.info-banner` CSS class:
```css
.info-banner {
    background: #dbeafe;
    border: 1px solid #93c5fd;
    color: #1e40af;
    padding: var(--space-xs) var(--space-md);
    border-radius: 6px;
    font-size: 14px;
    margin: 0;
}
```

Source: 03-CONTEXT.md D-05, matching sealed-badge color palette (#dbeafe / #1e40af).

#### 3. Two-step catalog picker in spool dialog

Location: In `#spool-dialog` `#spool-form`, replacing the current free-text `#spool-material` input. Inserted as the first two form-groups (above the Name field).

```html
<div class="form-group" id="catalog-material-group">
  <label for="spool-catalog-material">Material <span aria-hidden="true">*</span></label>
  <select id="spool-catalog-material" required>
    <option value="">— Select material —</option>
  </select>
</div>
<div class="form-group" id="catalog-color-group">
  <label for="spool-catalog-color">Color variant <span aria-hidden="true">*</span></label>
  <select id="spool-catalog-color" required disabled>
    <option value="">— Select material first —</option>
  </select>
</div>
```

Interaction contract:
- On page load: material select populated from `GET /api/catalog/materials`. Color select disabled until material chosen.
- On material change: rebuild color select from `GET /api/catalog/colors?material={m}`. Enable color select.
- On color selection: auto-fill `#spool-name` with `"${productTitle} — ${colorName}"`. Auto-fill `#spool-color-hex` and sync `#spool-color-picker` + `#spool-color-swatch`. Both fields stay editable.
- Edit/duplicate restore: pre-select material from saved spool.material, rebuild color select, pre-select color by matching productTitle + colorName pattern from spool.name.

The existing free-text `#spool-material` input (`<input type="text" id="spool-material">`) is **removed** from the dialog. The material value for the API payload is taken from the selected catalog entry, not a free-text field.

Source: 03-CONTEXT.md D-01, D-02, D-03, D-04; 03-CONTEXT.md Specific Ideas.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (sync) | "Sync Bambu catalog" |
| Sync in progress | "Syncing…" (button label while disabled) |
| Last synced — never synced | "Never" |
| Last synced — synced | ISO-local formatted timestamp, e.g. "2 May 2026, 14:32" — use `new Date(iso).toLocaleString('en-GB', {day:'numeric',month:'short',year:'numeric',hour:'2-digit',minute:'2-digit'})` |
| Last synced — sync failed | "Sync failed" (color: --color-destructive) |
| Pre-sync banner | "Sync the Bambu catalog first to enable adding spools." |
| Add Spool (disabled) | "Add Spool" — button text unchanged; disabled state communicated by opacity + cursor, not text change |
| Material select placeholder | "— Select material —" |
| Color select placeholder (awaiting material) | "— Select material first —" |
| Color select placeholder (material chosen, loading) | "Loading…" |
| Color select placeholder (material chosen, ready) | "— Select color —" |
| Dialog title — add | "Add Spool" |
| Dialog title — edit | "Edit Spool" |
| Dialog title — duplicate | "Duplicate Spool" |
| Empty state — no spools, no sync | heading: "No spools yet" / body: "Sync the Bambu catalog, then add your first spool." |
| Empty state — no spools, synced | heading: "No spools yet" / body: "Add your first spool to get started." |
| Error state — sync network failure | "Catalog sync failed. Check your internet connection and try again." (shown in stat area or a toast — see Interaction States) |
| Error state — save with no material | "Select a material to continue." |
| Error state — save with no color | "Select a color variant to continue." |

## Interaction States

### Sync button states

| State | Button text | Button disabled | Stat value | Stat color |
|-------|-------------|-----------------|------------|------------|
| Never synced | "Sync Bambu catalog" | false | "Never" | --color-muted |
| Syncing | "Syncing…" | true | "Syncing…" | --color-muted |
| Sync complete | "Sync Bambu catalog" | false | formatted timestamp | #1f2937 |
| Sync failed | "Sync Bambu catalog" | false | "Sync failed" | --color-destructive |

### Add Spool button states

| State | Button | Cursor | Banner visible |
|-------|--------|--------|----------------|
| Catalog empty (never synced) | disabled, opacity: 0.5 | not-allowed | yes |
| Catalog populated | enabled, full opacity | pointer | no |

Re-enable is reactive: after sync completion, JS removes `disabled` attribute and hides banner without page reload. Use `catalogCount > 0` check from polling completion or `GET /api/catalog/count`.

### Two-step picker in dialog

| State | Material select | Color select |
|-------|----------------|--------------|
| Dialog just opened (add) | placeholder selected | disabled, "— Select material first —" |
| Material selected, colors loading | value set | disabled, "Loading…" |
| Material selected, colors ready | value set | enabled, "— Select color —" |
| Color selected | value set | value set, name + color auto-filled |
| Edit/duplicate open | pre-selected | pre-selected, fields pre-filled |
