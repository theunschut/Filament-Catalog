## Component Inventory

### New: Duplicate Button

Placed immediately after the Edit button in each `.spool-row`. Inherits `.spool-edit-btn` class exactly — same height (implicit), padding (2px 8px), font-size (13px), border (1px solid --color-border), border-radius (4px), color (#374151).

```
[color swatch] [name/material] [owner] [weight] [price] [badge] [badge] [notes] [Edit] [Duplicate]
```

- Button label: `Duplicate`
- No icon — text label, matching `Edit`
- `type="button"` attribute mandatory
- `aria-label` not needed (label is already descriptive)
- On hover: `background: var(--color-bg)` — identical to `.spool-edit-btn:hover`

CSS addition to app.css: none required — the button reuses `.spool-edit-btn` class. If a distinct class is preferred for JS targeting, add `.spool-duplicate-btn` with identical property declarations.

### Modified: Spool Dialog (existing `#spool-dialog`)

When opened via Duplicate, the modal title changes to `Duplicate Spool`. All fields are pre-filled from the source spool using the existing `populateFormForEdit` logic pattern. The dialog operates in add mode (no `data-edit-id` set, Delete button hidden), so Save creates a new spool.

Modal title text:
- Add mode: `Add Spool` (unchanged)
- Edit mode: `Edit Spool` (unchanged)
- Duplicate mode: `Duplicate Spool` (new)

No structural HTML changes to the dialog are needed. The duplicate open function calls `resetFormForAdd()` then populates all fields from the source spool object, then calls `dialog.showModal()` — without setting `dialog.dataset.editId`.

## Interaction Contract

### Duplicate Button Click

1. User clicks `Duplicate` on a spool row.
2. `resetFormForAdd()` is called to clear any previous state.
3. `repopulateOwnerSelect(allOwners)` is called.
4. All fields are set from the source spool: name, material, colorHex, ownerId, weightGrams, pricePaid, spoolStatus, paymentStatus, notes.
5. `dialogTitle.textContent` is set to `'Duplicate Spool'`.
6. `dialog.dataset.editId` is left empty (or explicitly set to `''`).
7. Delete button (`#spool-delete-btn`) remains hidden.
8. `dialog.showModal()` is called.
9. User may edit any field freely.
10. Clicking `Save Spool` calls `createSpool(payload)` — identical to Add mode path.
11. On success: dialog closes, spool list refreshes (existing `close` event in app.js fires).
12. Original spool is unmodified.

### Cancel / Backdrop Click (no change)

Existing cancel and backdrop-click handlers are unchanged — they call `dialog.close()`.

### Save Button label

`Save Spool` — unchanged from existing Add and Edit modes. Do not change the save button label per mode.

### Cancel Button label

`Discard Changes` — unchanged from existing Add and Edit modes.
