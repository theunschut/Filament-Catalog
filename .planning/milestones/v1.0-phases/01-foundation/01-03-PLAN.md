  <action>
Create two files at the repo root (`D:/repos/Fillament Catalog/`):

1. Create `install.ps1`:
```powershell
#Requires -RunAsAdministrator
param(
    [string]$PublishDir = "$PSScriptRoot\publish"
)

$exePath = Join-Path $PublishDir "FilamentCatalog.exe"

if (-not (Test-Path $exePath)) {
    Write-Error "Executable not found at '$exePath'. Run 'dotnet publish' first."
    exit 1
}

New-Service -Name "FilamentCatalog" `
            -BinaryPathName $exePath `
            -DisplayName "Filament Catalog" `
            -Description "Local filament spool tracker" `
            -StartupType Automatic

Start-Service -Name "FilamentCatalog"
Write-Host "Service installed and started. Browse to http://localhost:5000"
```

Key rules:
- `#Requires -RunAsAdministrator` at line 1 — prevents silent failure when not elevated (per PATTERNS.md)
- `$PublishDir` defaults to `$PSScriptRoot\publish` — always points to published output, NEVER to `bin/Debug/` (avoids Pitfall 4 from RESEARCH.md)
- `Test-Path $exePath` guard gives a clear error message before attempting `New-Service`
- `StartupType Automatic` — fulfills INFRA-01 requirement that service auto-starts with Windows
- Service name `"FilamentCatalog"` MUST match the name in `AddWindowsService` in Program.cs

2. Create `uninstall.ps1`:
```powershell
#Requires -RunAsAdministrator

Stop-Service -Name "FilamentCatalog" -ErrorAction SilentlyContinue
Remove-Service -Name "FilamentCatalog"
Write-Host "Service removed."
```

Key rules:
- `-ErrorAction SilentlyContinue` on `Stop-Service` handles already-stopped service gracefully (no red errors if service isn't running)
- `Remove-Service` requires PowerShell 6+ or Windows PowerShell 5.1 with KB update; this is Windows 11 (PS 5.1 built-in) — if `Remove-Service` is unavailable at runtime, user can fall back to `sc.exe delete FilamentCatalog`

Do NOT include the uninstall logic inside install.ps1. Keep them separate.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Windows SCM → FilamentCatalog.exe | Service control manager starts/stops the process; binary path is validated by SCM |
| install.ps1 → New-Service | Script runs as Administrator; no external input beyond $PublishDir param |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Elevation of Privilege | install.ps1 running as Administrator | accept | Script is checked into repo and reviewed; `#Requires -RunAsAdministrator` makes the requirement explicit rather than silently failing |
| T-03-02 | Tampering | FilamentCatalog.exe binary in publish/ | accept | Local development machine; binary produced from reviewed source code in this repo |
| T-03-03 | Denial of Service | Service auto-starts consuming resources on boot | accept | Single lightweight ASP.NET Core process; local-only; no external load |
| T-03-04 | Spoofing | Another process binding port 5000 before service starts | accept | Local-only dev machine; no security boundary for port contention |
</threat_model>
