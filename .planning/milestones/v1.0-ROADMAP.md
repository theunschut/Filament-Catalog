# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-03
**Phases:** 1–6
**Total Plans:** 18

## Overview

Complete local-only filament inventory app running as a Windows service. Users can sync the Bambu Studio filament catalog from local installation files, log spools via a two-step material/color picker, track owners and payment status, and view a balance sidebar showing what each friend owes. The UI uses a collapsible owner-grouped tree view.

## Milestone Summary

**Key Decisions:**

- Web app over desktop (WPF/WinForms) — browser UI easier for data grids, modals, color swatches; Windows service makes it always-on
- Plain HTML/CSS/JS frontend — no build step, no framework overhead for a local-only tool
- Local filaments_color_codes.json over Shopify API — Cloudflare blocks .NET HttpClient to Bambu webshop; local Bambu Studio install has the data
- sc.exe delete for uninstall — Remove-Service requires PS 6+; Windows 11 ships PS 5.1 by default
- BackgroundService + Channel\<SyncJob\> capacity 1 DropNewest — simplest correct pattern for single-shot background work
- 202 + polling over SSE — simpler to implement and debug in plain HTML/JS
- [ApiController] MVC over minimal API — controller-based architecture is cleaner for a growing endpoint surface

**Issues Resolved:**

- Remove-Service unavailable on PS 5.1 — fixed with sc.exe delete in uninstall.ps1
- Cloudflare blocking .NET HttpClient to Bambu webshop — replaced with local filaments_color_codes.json reader
- Duplicate CSS selector for .btn-secondary — updated existing rule rather than appending duplicate

**Issues Deferred:**

- No milestone audit run before close (no /gsd-audit-milestone)
- Phase 2 progress table showed "In progress" despite completion (documentation inconsistency)
- REQUIREMENTS.md checkboxes partially unchecked (SYNC reqs only) despite all reqs being complete per traceability table

**Technical Debt Incurred:**

- No automated tests — all verification is human UAT + self-check in SUMMARY.md
- Spool catalog data is a denormalized copy at creation time — no FK to BambuProduct
